# Quiz - T-000

1. Why do we keep `.meteor/local/` out of version control?

<details>
<summary>Answer</summary>
It is machine-specific runtime/build state that is large, volatile, and not source code.
</details>

2. Which file controls the Meteor package list for this app?

<details>
<summary>Answer</summary>
`.meteor/packages`.
</details>

3. Where is Tailwind content scanning configured?

<details>
<summary>Answer</summary>
`tailwind.config.js` in the `content` array.
</details>

4. Which files initialize client and server entrypoints?

<details>
<summary>Answer</summary>
`client/main.js` and `server/main.js`.
</details>

5. Why is the startup hook in `imports/startup/server/index.js` intentionally minimal?

<details>
<summary>Answer</summary>
To keep a clean baseline and provide a clear extension point for later tasks.
</details>

6. What command should be used to install npm packages in a Meteor project?

<details>
<summary>Answer</summary>
`meteor npm install`.
</details>
