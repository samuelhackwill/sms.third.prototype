# Quiz - T-001

1. Why do we use a pull request template in this repo?

<details>
<summary>Answer</summary>
To enforce consistent PR quality, including summary, manual test steps, and completion checks.
</details>

2. Which three sections are required in our PR template?

<details>
<summary>Answer</summary>
Summary, How To Test Manually, and Checklist.
</details>

3. Why require manual test steps in each PR?

<details>
<summary>Answer</summary>
Because this project relies on operational verification and code review instead of a formal automated test suite.
</details>

4. What does requiring PRs before merge on `main` protect against?

<details>
<summary>Answer</summary>
Direct unreviewed changes landing on `main` without visibility or review.
</details>

5. Where should task-level quizzes be stored?

<details>
<summary>Answer</summary>
In `docs/quizzes/<task-id>.md`.
</details>

6. What is the expected review action at the end of each task PR?

<details>
<summary>Answer</summary>
Request a Codex code review.
</details>
