# Quiz - T-050B

1. Where is the Pixi stage lifecycle implemented for this task?

<details>
<summary>Answer</summary>
In `client/pages/stage.js`, via `createPixiStage({ mountEl })`.
</details>

2. Which methods does the stage factory expose?

<details>
<summary>Answer</summary>
`start()`, `stop()`, `destroy()`, `spawnMessages(messages)`, and `clear()`.
</details>

3. What lane formula is used for rendering bullets?

<details>
<summary>Answer</summary>
`LANE_HEIGHT = FONT_SIZE + LANE_PADDING`, and `laneCount = max(1, floor(height / LANE_HEIGHT))`.
</details>

4. What spawn position and speed range are used?

<details>
<summary>Answer</summary>
Spawn at `x = width + 20`, `y = laneIndex * LANE_HEIGHT`, with random speed in `[120, 380]` px/s.
</details>

5. How are bullets cleaned up to avoid memory growth?

<details>
<summary>Answer</summary>
When `x + widthPx < 0`, each bullet is removed from stage, destroyed, and deleted from the active map.
</details>

6. How does the stage react to admin messages from BroadcastChannel?

<details>
<summary>Answer</summary>
`{ type: "spawn", messages }` calls `spawnMessages(messages)`, and `{ type: "clear" }` calls `clear()`.
</details>
